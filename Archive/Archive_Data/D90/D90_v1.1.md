d&b audiotechnik은 D80(Phase 3-2)에서 브랜드 기본 우선순위("가장 상세하고 개정 이력 있는 문서 우선")가 확정되었다. D90은 D80과 달리 별도 데이터시트(SPS, 4p, 상세 스펙 표와 A&E 시방서까지 포함)가 확보되어, OM(Reference Manual)과 SPS 두 문서가 모두 상세하다 — 두 문서의 스펙 표 내용은 실질적으로 동일(SPS의 스펙 표가 OM Chapter 3 "Technical specifications"와 문구까지 일치)하여 충돌 없이 상호 보완적으로 사용했다. 출처 표기는 OM을 우선 기록하고 SPS를 병기한다(L-Acoustics의 "OM 최우선 브랜드 규칙"과는 별개로, 이번 소스 집합에서 두 문서가 사실상 동일 내용이라 실질적 우선순위 차이가 발생하지 않음).

본 파일은 SKILL v1.7의 "완벽한 양방향 스키마 동기화" 규칙에 따라 D80_v1.0.md, LA12X_v2.6.md, LA7.16_v1.4.md와 Key 목록을 100% 일치시킨 최초 개정본이다. D90 파싱 중 발견된 신규 속성은 [신규] 표시로, 기존 3개 파일에만 있던 Key 중 D90에 역주입된 Key는 [양방향 동기화] 표시로 구분했다. D80과 공유되는 d&b 고유 속성(CMRR, S/N ratio 등)은 기존 `d80_specific` 섹션에 값으로 채워 넣었고, D90에만 있는 완전 신규 개념(Milan 오디오 네트워크, EcoMode, Time to tone, AmpPresets 등)은 `d90_specific` 섹션을 신설하여 기록했다.

### 주석 및 출처 (Notes & Sources)

**출처**: OM p.8 "Technical specifications" (최우선) / SPS 2p / AE (txt) / WEB-F

**측정 조건**: 전 채널 구동. Amplification_Gain은 "Linear mode @ 0 dB" 조건(OM/SPS 일치). Output_Delay_Range는 채널별 독립 적용, 최대 10초(OM/SPS/AE 일치).

**[1] Rated_Power_Per_Ch 표 구조 (D80과 동일한 d&b 브랜드 고유 표기, 세대 차이 존재)**: D90의 OM/SPS/AE는 두 가지 조건의 채널당 출력을 명시한다 — (a) "Output power rating EIA-426B noise CF 12 dB": 8 Ω = 2700 W, 4 Ω = 5400 W(순간/버스트성 정격), (b) "Sine wave 1 kHz, long term, +40°C": 4 Ω = 450 W(연속 정격, 열 제한 조건). D80은 CF 6 dB/12 dB 두 조건을 병기했으나 D90은 EIA-426-B(노이즈 신호, CF 12 dB)와 1 kHz 사인파 장기 연속 조건을 병기하는 구조로 세대 간 표기 방식이 다르다 — 무결성 원칙에 따라 임의로 통일하지 않고 원문 그대로 "long term 값 (CF 12 dB 값)" 형식으로 병기했다. 8 Ω 조건의 long term 값은 소스에 4 Ω 조건(450 W)만 명시되어 있고 8 Ω 조건의 별도 long term 수치는 확인되지 않아, 8 Ω 행에도 동일하게 "450 (long term, 4 Ω 조건에서만 확인) / 2700 (CF 12 dB)"으로 병기하되 이 4 Ω/8 Ω 혼선을 각주에 명기한다. 상세는 아래 참조.

**CF 12 dB / long term 조건별 상세값** (OM p.8, SPS 2p, AE 일치): EIA-426B noise, CF 12 dB, 전 채널 구동 조건 — 4 Ω = 5400 W, 8 Ω = 2700 W. Sine wave 1 kHz, long term(연속), ambient +40°C 조건 — 4 Ω = 450 W(8 Ω 조건의 long term 수치는 소스에 별도 명시되지 않음). D80처럼 "Peak"라는 별도 명칭 대신 "EIA-426B CF 12 dB"(순간성 정격)와 "long term"(연속 정격, 열 제한)으로 조건이 구분되어 있어 `Peak_Power_Per_Ch_*` Key로 이관하지 않고 `Rated_Power_Per_Ch_*` 각주로 병기했다(D80 amplification 섹션 각주 [1]과 동일 처리 원칙).

**[2] Total_Power_Capability**: D90의 OM/SPS/AE 어디에도 전 채널 동시 구동 시 총 출력 용량을 나타내는 단일 수치가 명시되어 있지 않다. SPS p.2 "Peak output 3650 W"는 메인즈로부터 끌어오는 순간 입력 전력이며(power_supply 섹션 PSU_Mains_Rated_Power와 별개 개념, 아래 참조), 채널당 정격의 단순 합산은 임의 계산 금지 원칙에 따라 채택하지 않는다. null 처리.

### 주석 및 출처 (Notes & Sources)

**출처**: OM p.8 "Technical specifications", p.64-65 "10.1 Power supply" (최우선) / SPS 2p / AE (txt)

**측정 조건**: Power_Consumption_Idle/Standby는 OM/SPS 스펙 표 대표값(전압별 세부 구분 없음, D80과 달리 단일값으로만 제공). PSU_Type Value는 원문 "Switched mode power supply with automatic mains range selection and active Power Factor Correction (PFC)"를 그대로 채택 — D80의 "Autosensing" 대신 D90은 "automatic mains range selection"으로 표현하며, 두 표현이 동일 기능을 가리키는 것으로 판단되나 원문 그대로 보존했다.

**[3] Nominal_Current 값 산정**: OM/SPS "Rated mains current (High range) 16 A", "Rated mains current (Low range) 24 A"(OM p.8, SPS 2p, AE 일치: "24 A" 저전압/암시적 고전압). D80은 저전압대(100-120V)에서 30 A, D90은 24 A로 차이가 있다 — 제품 간 실제 사양 차이로 판단(충돌 아님).

**Power_Factor**: D90 소스(OM, SPS, AE) 어디에도 D80의 "above 0.9" 같은 역률 수치가 명시되어 있지 않아 null 처리했다. PFC 기능 자체는 탑재되어 있음(PSU_Type 참조)이 확인되나 구체적 역률 수치는 미확인이다.

**[4] PSU_Mains_Rated_Power 값 산정**: SPS p.2 "Peak output 3650 W"(OM p.8 동일 수치). D80의 "Max. power consumption (short term RMS) 7000 W"와 개념적으로 유사한 "순간 최대 메인즈 입력 전력"으로 판단하여 매핑했으나, D80은 "short term RMS"로 조건이 명시된 반면 D90은 "Peak output"이라는 다른 용어를 사용하여 정확히 동일한 측정 조건인지는 확정할 수 없다 — 원문 명칭(Peak output)과 수치만 채택하고 조건 차이는 각주로 남긴다.

### 주석 및 출처 (Notes & Sources)

**출처**: OM p.8 "Technical specifications" (최우선) / SPS 2p / AE (txt)

**측정 조건**: Frequency_Response는 -1 dB 허용오차, Linear mode 기준(OM/SPS/AE 일치). THD_N은 unweighted, 20-20 kHz 대역, "4 x 450 W" 조건(long term 정격 기준, OM/SPS/AE 일치). Channel_Separation(Crosstalk)은 20-20 kHz 대역, "4 x 450 W into 8/4 Ω" 조건.

**[5] Frequency_Response 상한 차이**: D90은 35-25000 Hz(OM/SPS/AE 일치)로, D80(35-20000 Hz)보다 상한이 확장되었다 — 세대 간 실제 사양 차이로 판단.

**[6] THD_N 표기 방식 차이 (D90 고유, dB와 % 병기)**: D80은 THD_N을 %(percent)로만 명시했으나(예: "less than 0.5%"), D90의 OM/SPS/AE는 "< -80 dB / 0.01%"(8 Ω), "< -74 dB / 0.02%"(4 Ω)로 dB와 % 두 단위를 함께 명시한다. 기존 Unit 컬럼이 "%"로 고정되어 있었으나 D90 원문이 dB를 주 표기로 사용하므로 Value에 dB값을 채택하고 %값은 괄호로 병기했다(Unit 컬럼도 dB로 조정) — 무결성 원칙(원문 표기 임의 변환 금지)에 따라 D90 원문의 주표기 방식을 그대로 반영한 것이며, D80/LA12X/LA7.16의 % 단독 표기와는 단위 표기 형식이 다를 수 있음을 명기한다. THD_N_8ohm/4ohm 두 조건 모두 D90은 값이 확인되나(D80은 THD_N_8ohm이 null이었음), D90은 "4 x 450 W" 동일 출력 조건에서 4/8 Ω 둘 다 측정했다는 점이 D80과 다르다(D80은 4 Ω, 600 W 조건 THD만 존재).

**[7] Channel_Separation 값**: OM/SPS/AE "Crosstalk (20-20 kHz) < -70 dBr"로 D80과 완전히 동일한 수치("less than -70 dBr")다. D80 각주([8])에서 설명한 부호/기준 차이(L-Acoustics의 상대적 "greater than" 표기와 반대 방향)가 D90에도 동일하게 적용되며, 원문 그대로 보존했다.

**Output_Dynamic_Range, Noise_Level, Damping_Factor**: D90 소스에는 D80과 마찬가지로 이 세 Key에 정확히 대응하는 스펙이 확인되지 않는다(Damping_Factor는 D80에는 있었으나 D90 소스 어디에도 명시되지 않아 null). D90의 "Output noise/Dynamic range" 절 수치(Analog/Digital input 기준 dB, dynamic range)는 D80과 마찬가지로 신호 대비 상대값이 아닌 절대 노이즈 전압(uV RMS)과 그에 대응하는 dB 수치이며, 물리량 기준이 기존 Key와 다르다고 판단하여 아래 [신규] Key(Output_Noise_Dynamic_Range_Detail, d90_specific 섹션)로 별도 기록했다.

### 주석 및 출처 (Notes & Sources)

**출처**: OM p.8 "Technical specifications", p.57 "8.8.2 LoadMatch", p.41-43 "7.5.4-7.5.6" (최우선) / SPS 2p / AE (txt)

**측정 조건**: 없음(사양값). DSP_Processor는 D80과 달리 D90 소스(OM p.8 "Internal processing")에 명시적으로 기재되어 있어 D80(null)과 다르게 값이 채워졌다 — 구체적 칩셋명이 아닌 처리 방식(fixed point + floating point 조합) 서술이다.

**[8] Amplifier_Power_Management 값 확장**: D80은 MCL만 확인되었으나, D90은 MCL(OM p.41 "7.5.4 Mains current limiter", 95-50% 범위, 5% 단위)에 더해 EcoMode(OM p.42 "7.5.5 Energy saving" — 출력 레벨이 낮을 때 60초 카운트다운 후 저전력 모드 자동 전환, 고전압 185V/저전압 95V 미만에서는 비활성)라는 별도 명명된 절전 기능이 추가로 확인되어 함께 병기했다. EcoMode 상세 동작 로직은 아래 [신규] Key(Energy_Saving_Detail, d90_specific 섹션)에 별도 기록한다.

**[9] Speaker_Protection**: D90의 LoadMatch(OM p.57, "up to 70 m", 20 kHz 대역폭)는 D80과 동일한 기능명·조건이다. 다만 D80에 병기되어 있던 "ArrayProcessing-compatible" 서술은 D90 소스(OM, SPS, AE) 어디에도 확인되지 않아 D90에는 포함하지 않았다(제품별 실제 차이로 판단, 임의 이식 금지 원칙).

**[신규][9a] DSP_Sampling_Rate 값의 성립 조건 및 D80과의 서술 유무 차이 (양방향 동기화 보강)**: D90의 74페이지 Reference Manual 전문을 "sampl", "SRC", "Conversion", "clock" 키워드로 전수 검색한 결과, D80 OM p.34 "System clocking"과 같은 조건부 SRC 활성화/비활성화 서술이나 내부 클록 도출 메커니즘 설명이 전혀 확인되지 않았다. D90 소스는 "permanent, high-speed, high-quality SRCs"(상시 동작 고품질 SRC)라는 표현만 있을 뿐, D80처럼 "기본적으로 SRC를 사용하지 않다가 특정 조건에서만 수동 활성화"하는 조건부 구조에 대한 설명이 없다. 즉 D90의 SRC는 항상 켜져 있는 것으로 서술되어 있어(D80과 반대 방향의 아키텍처), DSP_Sampling_Rate(96 kHz)는 조건에 관계없이 성립하는 값으로 판단된다. **D80과의 서술 유무 차이**: D80의 OM에는 내부 클록이 96 kHz로 고정되고 48 kHz 입력이 자동 업샘플링된다는 상세 설명(p.34)이 있는 반면, D90의 Reference Manual에는 이 수준의 클로킹 동작 원리 설명이 없다 — SRC가 "permanent"라는 단어로만 언급될 뿐 그 근거가 되는 내부 동작 메커니즘 서술은 확인되지 않는다. 이는 D90 소스 문서의 상세도가 D80보다 낮다는 의미이며, 값의 신뢰도 차이가 아니라 문서화 수준의 차이임을 명시한다.

### 주석 및 출처 (Notes & Sources)

**출처**: OM p.8 "Technical specifications" (최우선) / SPS 2p / AE (txt)

**측정 조건**: OM/SPS "Latency analog/digital (AES) input 0.3/0.3 msec" — 아날로그/디지털 입력 모두 동일 수치(D80과 동일 패턴). Latency_Low_Mode는 D80과 마찬가지로 별도 "Low latency 모드" 개념이 확인되지 않아 비적용(null). AVB_Max_Network_Latency는 D90이 AVB가 아닌 Milan(별도 프로토콜, 아래 input_avb 섹션 참조)을 사용하므로 비적용(null) 처리했다 — Milan 자체의 네트워크 지연시간 수치는 D90 소스에 별도로 명시되지 않았다.

**[신규][9b] Latency_Standard 값의 성립 조건 및 D80/LA12X/LA7.16과의 서술 유무 차이**: D90 소스(OM 74p, SPS)에는 "permanent, high-speed, high-quality SRCs"라는 표현이 있어 D80과 달리 SRC 조건부 활성화에 따른 레이턴시 변동 서술이 없다 — 즉 D90의 0.3 ms는 D80처럼 "SRC off, 48/96 kHz" 조건이 붙는 값이 아니라 무조건적으로 적용되는 값으로 보인다. 다만 이는 D80의 OM처럼 "SRC가 상시 동작하며 조건에 관계없이 일정한 지연을 보장한다"는 명시적 확정 문구(LA12X/LA7.16의 "independent from input Fs"에 해당하는 표현)가 D90 소스에 존재하지 않는다는 한계가 있다 — SRC가 "permanent"라는 단어로만 서술될 뿐, 레이턴시가 입력 샘플레이트와 무관함을 직접 명시하는 문장은 확인되지 않았다. **양방향 참고**: LA12X(OM p.86)/LA7.16(OM p.94)은 "independent from input Fs"라는 확정 문구가 스펙 표에 직접 붙어 있어 근거 수준이 D90보다 높고, D80은 반대로 조건부 SRC 활성화 서술(OM p.34)이 있어 오히려 값이 무조건적이지 않음을 스스로 밝히고 있다. D90은 이 세 제품 중 가장 서술이 간략하여, "SRC가 항상 켜져 있다"는 사실은 확인되나 그로부터 "레이턴시가 항상 일정하다"는 결론까지 소스가 명시적으로 잇지는 않는다는 점을 이 각주에 기록한다.

**[22] Depth 소스 간 충돌**: OM(p.8), SPS(3p), WEB-F는 모두 "512 mm/20.2""로 일치한다. 그러나 AE(txt, v1.2)는 동일 항목을 "465 mm (18.3")"로 명시하여 다른 세 소스와 불일치한다. 두 값 모두 보존 — 채택값 512 mm는 3개 독립 소스(OM, SPS, WEB-F) 일치를 근거로 하며, AE의 465 mm는 단일 소스 이례치로 오기 가능성이 있으나 원문 값이므로 삭제하지 않는다. 참고로 D80의 Depth(530.5 mm)와 D90의 465 mm는 우연히도 비슷한 범위이나 이는 채택하지 않은 값이므로 근거로 사용하지 않는다.

**Protection_Rating, Finish**: D90 소스 어디에도 명시되지 않아 null 처리했다(D80과 동일 패턴).

**[24][양방향 동기화 참고] SN_Ratio_Analog/Digital_Input**: 이 Key는 D80에서 "S/N ratio (unweighted, RMS): Analog input > 110 dBr, Digital input > 114 dBr" 형태로 신설되었으나, D90 소스에는 이와 동일한 형식(단순 dBr 상대값)의 S/N ratio 항목이 확인되지 않는다. 대신 D90은 "Output noise/Dynamic range" 절에서 절대 노이즈 전압(uV RMS)과 그에 대응하는 dB 수치를 A-weighting 여부별로 4가지 조건(Analog input, Analog input A-weighted, Digital/Milan input, Digital/Milan input A-weighted)으로 상세하게 제공한다 — 이는 D80의 단순 SN_Ratio_*_Input Key보다 훨씬 상세하고 다른 구조의 데이터이므로, 억지로 이 Key에 채워 넣지 않고 SN_Ratio_Analog/Digital_Input은 null로 유지하며 D90의 실제 값은 아래 d90_specific 섹션의 Output_Noise_Dynamic_Range_Detail Key에 전문 기록했다.

**Load_Monitoring_System_Check, Signal_Path_Management, LoadMatch_Max_Cable_Length, Output_Mode_Selectable, Mains_Current_Limiter_Range**: D90 OM/SPS/AE에 D80과 개념적으로 동일한 기능이 확인되어 D90 고유의 세부 차이(위 각주 참조)를 반영해 값을 채택했다. Output_Mode_Selectable에서 D80에 있던 "LINEAR mode" 문구는 D90 스펙 표에서 별도로 재확인되지 않아(OM Chapter 1 "Intended use"에는 LINEAR setup 언급이 있으나 Output mode 선택지 자체로는 명시 안 됨) 제외했다 — 값 손실이 아니라 D90 소스 구조상 위치가 다름을 반영한 것이며, LINEAR 모드 자체는 OM p.6에서 별도로 확인된다(추후 정밀 검토 대상).

## 버전 변경 이력

v1.0: Phase 3-1 동일 브랜드(d&b audiotechnik) 이기종 모델 최초 파싱. OM(Reference Manual 1.3, 74p), SPS(Datasheet 1.4, 4p), AE(txt, 1.2), 웹 3계층 총 6개 소스를 교차 검증하여 마스터 스키마 최초 구축. D80과 공유되는 d80_specific 섹션 12개 Key 중 8개(CMRR 확장, System_Start_Up_Time, Storage_Temp/Humidity, LoadMatch_Max_Cable_Length, Signal_Path_Management, Load_Monitoring_System_Check, Output_Mode_Selectable, Mains_Current_Limiter_Range)에 값을 채우고 SN_Ratio_*는 D90 고유 상세 데이터 구조 차이로 null 유지. D90 고유의 완전 신규 속성 9개(Milan 프로토콜 상세, Time to tone, Energy saving 상세, AmpPresets 상세, Output noise 상세)를 d90_specific 섹션으로 신설하여 D80/LA12X/LA7.16 3개 파일에 null로 소급 반영. AE(txt)의 Depth(465 mm)가 OM/SPS/WEB-F(512 mm)와 충돌하는 것을 발견하여 각주로 양쪽 보존.

v1.0 to v1.1: (사용자 검토 반영, D80 라운드에서 발견된 이슈의 전 제품 확장 적용) D80에서 DSP_Sampling_Rate/Latency_Standard 값의 성립 조건 근거 부족이 지적되어, 동일 브랜드인 D90도 재검토했다. D90의 74페이지 Reference Manual 전문을 "sampl/SRC/Conversion/clock" 키워드로 전수 검색한 결과, D80과 같은 조건부 SRC 활성화 서술은 없으며 "permanent, high-speed, high-quality SRCs"(상시 동작)로만 서술되어 있음을 확인했다. 이에 따라 dsp 섹션 DSP_Sampling_Rate와 latency 섹션 Latency_Standard에 각주 [9a][9b]를 신설하여, D90 소스의 서술 상세도가 D80(조건부 SRC 메커니즘 상세 설명 있음)이나 LA12X/LA7.16("independent from input Fs" 확정 문구 있음)보다 낮다는 점 — 즉 SRC가 상시 동작한다는 사실은 확인되나 이로부터 레이턴시 불변성을 직접 명시하는 문장은 소스에 없다는 한계 — 을 양방향 동기화 관점에서 명기했다. Value 자체(96, 0.3)는 변경하지 않았다.

**파일 무결성 참고**: 본 파일은 세션 중 발생한 파일 저장 동기화 지연으로 인해 v1.1 최초 저장 시 이 지점(d90_specific 섹션 각주 [26] 중간)에서 내용이 잘려 저장되었던 것을 확인하여, Archive의 D90_v1.0.md 원문을 대조해 [27] 각주, Null Report, 버전 이력을 복원했다. 표 데이터(Key/Value/Unit)와 dsp/latency 섹션 각주([9a][9b] 포함)는 영향받지 않았다.
