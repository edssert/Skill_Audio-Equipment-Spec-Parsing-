# Session Transfer — 음향 기기 데이터 파싱 스킬 프로젝트

## 프로젝트 개요

목표는 음향 기기(앰프, 스피커, 컨트롤러 등) 도메인에 특화된 데이터 파싱/스키마 통합 스킬을 스킬 크리에이터 방식으로 반복 테스트하며 구축하는 것이다. 구동 대상은 주력 Claude Sonnet(대량 반복 파싱), 보조 Claude Opus(복잡한 매뉴얼 분석·스키마 충돌 해결)이다. 지침 문서 구조는 마크다운 골격 + XML 국소 삽입 하이브리드로 확정했다(스킬 포맷 요건 충족 + 예시/데이터 블록의 경계 모호성 원천 차단).

## 완료된 작업 (Phase 1-2)

- Phase 1-2: L-Acoustics LA12X(4채널 앰프)를 대상으로 오너 매뉴얼 PDF, 스펙시트 PDF, A&E 시방서 docx, 웹사이트 3계층(리스트뷰/오버뷰/Full Spec) 총 6개 소스를 교차 검증하여 단일 마스터 스키마 마크다운으로 병합 완료.
- 병합 과정에서 8건의 소스 간 데이터 충돌을 발견하고 전부 기록(양쪽 값 보존 원칙 적용).
- 사용자 피드백을 거쳐 총 4회 리팩토링(v1.0 -> v4.0)했으며, 각 회차마다 스킬 지침(SKILL.md) 자체도 함께 갱신했다.
- 첨부된 LA12X_v1.0.md ~ LA12X_latest.md(v4.0) 파일이 각 리팩토링 단계의 실제 결과물이다.

## 고도화된 핵심 스킬 지침 요약 (현재 SKILL_v1.4_latest.md 기준)

1. **섹션 단위 출처 표기**: 개별 셀 각주 대신 섹션 하단 "주석 및 출처" 섹션에 문서+페이지/목차를 한 번에 서술.
2. **표의 순수성 유지**: 표 컬럼은 Key/Value/Unit만. 측정 조건·출처·채택 근거는 표 밖 주석으로. 각주는 충돌 등 예외적 개별 값에만.
3. **데이터 충돌 보존**: 소스 간 값이 다르면 절대 한쪽을 삭제·평균 내지 않는다. Value에는 채택값, 각주에 양쪽 원문 값과 채택 근거를 병기.
4. **L-Acoustics 브랜드 출처 우선순위**: 오너 매뉴얼(OM)의 페이지/목차 위치를 항상 최우선으로 기록.
5. **이모지 및 특수 아이콘 엄격 금지**: 지침 문서와 결과물 어디에도 이모지·장식 아이콘 사용 불가.
6. **저항값(Impedance) 표기 표준화**: 유럽식 표기(2R7, 4R, 8R)를 소수점(2.7, 4, 8)으로 정제. Value뿐 아니라 동적 생성 속성명(Key)에도 동일 적용 — 예: `Rated_Power_Per_Ch_2R7` -> `Rated_Power_Per_Ch_2.7_Ohms`.
7. **임의 추론/수식어 첨가 절대 금지**: 원본에 없는 수식어·마케팅 문구·괄호 설명을 Value에 덧붙이지 않는다. 소스별 표현이 다르면 조용히 병합하지 말고 반드시 충돌 보존 원칙(각주)에 따라 처리한다.
8. **Phase 3 세분화 및 하위 호환 동기화 (신규, 아직 미실행)**: Phase 3-1(동종 브랜드 이기종 채널)과 Phase 3-2(타 브랜드 교차)로 나뉜다. 신규 속성 발견 시 반드시 기존 제품 파일 전체에 해당 Key를 null로 소급 반영하고, 갱신된 기존 파일들을 신규 제품 파일과 함께 일괄 재출력해야 한다(Phase 3-1은 2개 파일, Phase 3-2는 3개 파일 동시 출력이 필수 요건).

동봉된 SKILL_v1.0.md ~ SKILL_v1.4_latest.md는 각 규칙이 추가된 시점별 지침 전문이다. 단, v1.0/v1.1은 실제 파일 저장 시점 기준으로는 최초 SKILL.md 생성 시 이미 v1.1 내용까지 함께 작성되었던 것을 규칙 도입 순서에 맞춰 소급 재구성한 버전이며, v1.2 이후부터는 실제 편집 이력과 완전히 일치한다.

## 다음 세션에서 즉시 진행할 작업: Phase 3-1

**목표**: L-Acoustics 16채널 라인업(LA7.16(i) 또는 LA1.16i 등)의 스펙 데이터를 투입받아 파싱한다.

**구체적 지시사항**:

1. 사용자가 LA7.16(i) 등 16채널 앰프의 데이터(매뉴얼 PDF, 스펙시트, 웹 데이터 등)를 제공하면, 위 스킬 지침 8개 원칙을 전부 적용하여 신규 제품의 마스터 스키마 마크다운을 작성한다.
2. 16채널 아키텍처로 인해 LA12X(4채널)에는 없던 새로운 속성(Key)이 발견될 것이 확실하다 — 예를 들어 채널 그룹핑, 다중 출력 매핑, TFT 터치스크린 관련 속성 등. 이런 신규 Key가 발견되면 예외 없이 LA12X_latest.md(v4.0)를 읽어와 해당 Key를 `null` 값으로 추가하고 리팩토링한다.
3. **결과물은 반드시 2개 파일**로 출력한다: (a) 신규 LA7.16 마스터 스키마 파일, (b) 신규 Key가 null로 반영되어 갱신된 LA12X 마스터 스키마 파일(버전 번호를 v5.0으로 올려서 저장). 신규 제품 파일 1개만 출력하고 끝내는 것은 지침 위반이다.
4. 리팩토링 후에는 지침(SKILL.md)에도 이번 Phase 3-1에서 새로 확정된 규칙(예: 채널 수가 다른 제품 간 스키마 확장 패턴)이 있다면 반영하고, 최신 지침 전문을 코드 블록으로 함께 출력한다(사용자가 이전에 요청한 "결과물 + 지침 전문 동시 출력" 운영 규칙).
5. 이후 Phase 3-2(d&b audiotechnik D80/D90 등 타 브랜드)가 예정되어 있으며, 그 단계에서는 LA12X + LA7.16 갱신본 2개 + d&b 신규 1개 = 총 3개 파일을 동시 출력해야 한다.
