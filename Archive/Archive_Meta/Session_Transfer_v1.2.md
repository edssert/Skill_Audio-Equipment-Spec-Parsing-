# Session Transfer — 음향 기기 데이터 파싱 스킬 프로젝트 (v1.2)

## 프로젝트 개요

L-Acoustics, d&b audiotechnik 등 여러 브랜드의 음향 기기 스펙 데이터를 오너 매뉴얼/스펙시트/A&E 시방서/웹 3계층에서 교차 검증해 단일 마스터 스키마 마크다운으로 통합하는 스킬 크리에이터 프로젝트. 상세 파싱 규칙 전문은 항상 루트의 최신 `SKILL_vX.X.md` 1개를 참조할 것 — 본 파일에는 규칙을 반복하지 않는다.

## 현재 상태: Phase 3-2 완료, Phase 4 전환 대기 (누적 제품 3개, 전환 기준 5개 미달)

- Phase 1-2: LA12X(4채널) 6개 소스 교차 검증, 마스터 스키마 최초 구축 완료.
- Phase 3-1: LA7.16(16채널) 신규 파싱 + LA12X와 완벽한 양방향 스키마 동기화(Key 목록 100% 일치) 완료.
- Phase 3-2: d&b audiotechnik D80(4채널) 신규 브랜드 최초 투입 완료. OM(1.14판, 93p)과 A&E 시방서(txt), 웹 3계층(총 5개 소스)을 교차 검증하여 D80 마스터 스키마를 최초 구축했다. 기존 두 제품(LA12X/LA7.16) 스키마와 개념이 정확히 대응되지 않는 12개 D80 고유 속성(CMRR, S/N ratio, LoadMatch 케이블 보상, 선택형 출력 모드 등)을 발견하여 신규 섹션 `d80_specific`으로 분리 신설했고, 이 섹션을 LA12X/LA7.16 양쪽에도 null로 소급 반영했다. 반대로 LA12X/LA7.16 전용 Key(예: AVB 관련 8개, SC32/speakON 커넥터 등) 21건은 D80에 "비적용(null)"으로, 소스 미확인 24건은 D80에 "미확인(null)"으로 구분 반영했다. 3개 파일(LA12X_v2.6, LA7.16_v1.4, D80_v1.0)의 Key 목록이 100% 동일함을 파이썬 스크립트로 검증 완료(14개 섹션, 112개 Key 일치).
- SKILL.md v1.9 갱신: d&b audiotechnik의 출처 우선순위를 "기본값(가장 상세한 문서 우선)" 적용 사례로 명문화. 무결성 원칙에 "표기 기준이 다른 동일 물리량 임의 변환 금지"(D80 Crosstalk vs LA Channel_Separation 부호 차이 사례), "확정된 비존재(0)와 미확인(null) 구분"(D80 GPIO 전문 검색 0건 -> GPIO_Count=0 사례) 조항 추가.
- 작업공간 정리: 시맨틱 버저닝 유지, 구버전(SKILL_v1.8, LA12X_v2.5, LA7.16_v1.3, Session_Transfer_v1.1) Archive로 이관.

**현재 루트 최신 파일** (다음 세션은 이 4개 + 아래 폴더만 확인하면 됨):
- `SKILL_v1.9.md` — 파싱 규칙 전문(양방향 동기화, PDF 분석 규칙, 원본 보존 규칙, 버전 체계화 규칙, 브랜드별 우선순위, 표기 기준 차이/확정적 비존재 처리 규칙 등 전부 포함)
- `LA7.16_v1.4.md`, `LA12X_v2.6.md`, `D80_v1.0.md` — 세 제품 마스터 스키마(Key 목록 100% 동일, 14개 섹션 112개 Key)
- `Original_PDFs/` — 세 제품 원본 매뉴얼/스펙시트/A&E 문서 보존 (D80은 OM만 보유, 별도 SPS 없음)
- `Raw_Web_Data/` — 세 제품 웹 텍스트(리스트뷰/오버뷰/Full Spec) 원문 및 D80의 A&E(txt) 원문 무수정 보존
- `Archive/` — 모든 구버전(과거 판단 근거 확인용, 평소엔 참조 불필요)

## 다음 세션에서 진행할 작업: Phase 3-3 (선택) 또는 Phase 4 준비

**옵션 A — Phase 3-3 계속**: 세 번째 브랜드(예: Powersoft, Lab.gruppen, Crown 등)를 추가 투입하여 용어 매핑 사전과 브랜드 우선순위 규칙을 더 확장한다. 이 경우 Phase 3-2와 동일한 절차(신규 브랜드 파싱 -> 양방향 동기화 -> 3개 이상 기존 파일 전체 갱신)를 반복한다.

**옵션 B — Phase 4 전환 준비**: 누적 제품 수가 5개에 도달하면(현재 3개), 매번 전량 재출력 대신 "변경된 Key와 영향받는 파일 목록"을 먼저 요약하고 사용자에게 일괄/개별 출력 여부를 묻는 방식으로 전환한다(SKILL v1.9 Phase 4 절 참조). 아직 기준 미달이므로 신규 제품 1-2개를 더 투입한 뒤 전환 검토.

**필수 요건 공통** (SKILL_v1.9.md 참조):
1. 신규 제품에서 발견되는 신규 Key(신규 섹션 포함)는 기존 전 제품 파일에 `null`로 소급 반영(양방향 동기화 — 반대로 기존 제품 전용 Key도 신규 제품에 `null`로 반영).
2. 결과물은 관련된 전체 제품 파일을 동시 출력. N개 제품이면 N개 파일의 Key 목록이 100% 동일함을 스크립트로 검증할 것.
3. 원본 문서는 `Original_PDFs/`에, 웹 원문(및 txt로 제공되는 A&E 등 비-PDF 원문)은 `Raw_Web_Data/`에 동일 컨벤션으로 보존.
4. 새 파일 생성만 허용, 기존 파일 직접 수정 금지 — 버전을 올려 새로 만들고 구버전은 `Archive/Archive_Data`(제품 스키마) 또는 `Archive/Archive_Skill`(SKILL.md)로 이동.
5. L-Acoustics 브랜드 우선순위 규칙(OM 최우선)은 L-Acoustics 제품에만 적용. d&b audiotechnik은 기본값("가장 상세하고 개정 이력 있는 문서 우선") 적용이 Phase 3-2에서 확정됨. 신규 브랜드 투입 시 동일하게 기본값을 적용하되, 그 브랜드에 고유한 문서 위계가 뚜렷하면 SKILL.md에 전용 절을 신설할 수 있음.

## 참고: 과거 이력 요약 (상세 근거는 Archive 참조)

- SKILL.md는 v1.0부터 v1.9까지 9회 개정(각주 체계, 이모지 금지, 저항 표기 표준화, Phase 3 세분화, PDF 분석 규칙, 원본 보존 규칙, 양방향 동기화, 버전 체계화, d&b 브랜드 우선순위 확정/표기 차이 처리 순으로 누적). 전문은 `Archive/Archive_Skill/`.
- LA12X는 원래 v1.0~v5.1로 발행되었으나 시맨틱 버저닝 소급 적용으로 v1.0/v2.0/v2.1/v2.2/v2.3/v2.4/v2.5/v2.6으로 재분류/누적됨(v2.0만 Major, 이후 전부 Minor). 재분류 근거는 `Archive/Archive_Data/`의 각 파일 상단 "버전 재분류 안내" 참조.
- LA7.16은 v1.0(최초 파싱, Total_Power_Capability 오기재 있었음) -> v1.1(양방향 동기화 + 오기재 정정) -> v1.2(LA12X 파일명 참조 갱신) -> v1.3(12-point terminal block 핀맵 보강) -> v1.4(Phase 3-2, d80_specific 섹션 null 반영) 순으로 발행. 전문은 `Archive/Archive_Data/`.
- D80은 v1.0(Phase 3-2 최초 파싱, LA12X/LA7.16과 양방향 동기화 동시 반영) 단일 버전으로 시작. 이전 버전 없음.
- 이 파일(Session_Transfer.md) 자체도 v1.0(Phase 1-2 시점 원본), v1.1(Phase 3-1 완료 시점)을 `Archive/Archive_Meta/`에 보존하고 있다.
