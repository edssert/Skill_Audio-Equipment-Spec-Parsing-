**스켈레톤 재사용 근거 (SKILL v1.11 "신규 제품 스켈레톤 재사용 규칙")**: 본 파일은 D80_v2.0.md의 섹션 구조를 뼈대로 복제한 뒤 서지컬 교체하여 작성되었다(기존 11개 파일 중 어느 것도 5D의 Dante 아키텍처와 정확히 일치하지 않아 임의로 D80을 선택 — D80이 Milan/터치스크린 계열 중에서는 그나마 CAN-Bus/OCA 기반의 비-Milan 제어 구조를 가져 5D의 "Milan 완전 미사용" 특성과 구조적으로 가장 마찰이 적다고 판단). 다만 실제 Value 대부분은 D80과도 크게 다르다 — 5D는 (1) Milan/AVB 대신 Dante 네트워크 오디오를 사용하고, (2) 커넥터 체계가 XLR/NL4/NL8이 아닌 전량 Euroblock(아날로그 입력, 스피커 출력, GPI, FAULT 전용)이며, (3) 사용자 인터페이스가 터치스크린/LCD가 아닌 LED 표시등뿐이고, (4) 19인치가 아닌 9.5인치 하프랙(1U) 폼팩터이고, (5) 설치용 저출력(채널당 정격 75W급, 투어링 라인의 250~450W급과 확연히 다름)이다.

**중요 — 소스 제약 사항**: 이번 소스 집합은 데이터시트(SPS, 4p)와 스타트업 매뉴얼(OM, "Start-up manual", 23p)로 구성되며, D80/D90/D25/D40에 제공되었던 것과 같은 완전판 Reference Manual이 아니다. OM 자신도 "A full version of this manual (Reference manual) with comprehensive information is available for download..."라고 명시하여, 이 스타트업 매뉴얼이 축약판임을 밝히고 있다. 따라서 Mains Current Limiter(MCL)/Energy saving(EcoMode)/AmpPresets 등 D25/D40/D80/D90에 있던 상세 기능 챕터 자체가 이 소스 집합에 없다 — 이는 5D에 해당 기능이 "확정적으로 없다"는 의미가 아니라 "이번 소스로는 확인 불가"임을 뜻하므로, 관련 Key는 비적용(No/0)이 아닌 미확인(null)으로 보수적으로 처리했다. 웹 3계층 텍스트도 제공되지 않아 Raw_Web_Data는 생성하지 않았다.

**신규 스키마 확장(v1.0)**: 5D 파싱 중 기존 스키마 어디에도 대응 개념이 없는 두 가지가 발견되어 아래와 같이 반영했다 — (1) **input_dante 신규 섹션**: Dante는 input_avb 섹션이 다루는 Milan-AVB/Milan Endstation과는 별개의 오디오오버이더넷 생태계(Audinate Dante)이므로, 기존 input_avb에 억지로 끼워 넣지 않고 새 섹션을 신설했다(SKILL "스키마 동적 발견" 원칙). (2) **connectivity 섹션의 Fault_Contact_Type 신규 Key**: 원격 감시용 FAULT 릴레이 접점(NO/C/NC)은 기존 스키마 어디에도 대응 Key가 없어 신설했다. 두 확장 모두 기존 11개 파일(D80, D90, D25, D40, LA12X, LA7.16, LA1.16i, LA2Xi, LA4X, LA7.16i)에 양방향 동기화(null)가 필요하다(세션 후속 조치 대상).

### 주석 및 출처 (Notes & Sources)

**출처**: OM p.8 "3 Technical specifications" (최우선) / SPS 2p

**측정 조건**: 전 채널 구동. Amplification_Gain은 "Linear mode @ 0 dB" 조건(OM/SPS 일치).

**[1] Rated_Power_Per_Ch 표 구조 (D25/D40/D80/D90과 동일한 CF12dB+long term 병기 방식)**: 5D의 OM/SPS는 "Output power rating EIA-426B noise CF 12 dB: 4 x 600 W/8, 4 x 600 W/4"(즉 8 Ω과 4 Ω 모두 600 W로 동일 — 투어링 라인과 달리 저부하에서 전력이 늘지 않는 설치용 정전력 설계로 판단)와 "Sinus 1 kHz, long term, +40°C: 4 x 37.5 W/4"를 명시한다. 8 Ω 조건의 long term 수치는 소스에 별도로 명시되지 않아, D25/D40과 동일한 패턴으로 4 Ω 조건에서만 확인된 값(37.5W)을 8 Ω 행에도 병기하고 이 제약을 각주로 명기했다(CF 12dB 값은 8Ω/4Ω 실측치가 각각 명시되어 있어 그대로 채택). OM 3.1절의 CF 9dB/6dB 상세값(참고용, 표 미채택): CF 9dB(8Ω=350W,4Ω=350W), CF 6dB(8Ω=175W,4Ω=175W) — 전 부하조건에서 8Ω=4Ω 동일값 패턴이 CF 조건 전반에 반복된다.

**[2] Total_Power_Capability**: 5D의 OM/SPS 어디에도 전 채널 동시 구동 시 총 출력 용량을 나타내는 단일 수치가 명시되어 있지 않다. null 처리(임의 계산 금지 원칙).

**[3] Output_Delay_Range**: OM/SPS "Delay 1.1 - 300 ms(100 m/328 ft)" — D25/D40/D80/D90(0.3-10000ms, 10초/3440m)과 크게 다른 범위. 하한이 5D 자신의 Latency_Standard(1.1ms)와 일치하는 것은 지연시간이 딜레이 최소 단위로도 기능함을 시사한다(D25/D40/D80/D90도 동일 패턴: 하한=Latency_Standard).

**[4] Bridging_Support**: OM Chapter 5.2.4 "OUTPUTS"에 "Bridge mode is not applicable."이라는 명시적 확정 문구가 있다 — LA4X/LA7.16i와 동일한 수준의 확정적 비존재(No)로 판단, D25/D40/D80/D90(원본에 언급 자체가 없어 null 유지)과 달리 5D는 명확한 부인 문구가 있어 No로 명시한다. BTL/PBTL 관련 4개 전력 Key는 그 동작 모드 자체가 없어 측정값이 정의되지 않으므로 null 유지(LA4X 선례 준용).

### 주석 및 출처 (Notes & Sources)

**출처**: OM p.8 "3 Technical specifications", p.15 "5.2.1 Mains connection" (최우선) / SPS 2p

**측정 조건**: Power_Consumption 값은 OM Chapter 3.1의 "230 VAC/50 Hz" 상태별 상세 전류표(Standby/AutoWakeup/Idling, 각 상태의 Mains current[A]/Power factor/Input power[W])를 채택 — 산술 검증: Standby(0.1A x 0.17 x 230V = 3.9W ≈ 4W 일치), Idling(0.4A x 0.6 x 230V = 55.2W, 표기값 49W와 다소 차이 있으나 근사, 반올림/측정오차 범위로 판단). D25/D40과 달리 5D는 표 구조가 단순(Off 상태 행 없음, AutoWakeup 상태가 별도로 명시됨 — ReadyStandby/Eco 개념 없음).

**[1] Mains_Voltage_Range**: 5D는 D25/D40/D80/D90의 "고전압대/저전압대 분리"(예: 100-127/208-240) 방식이 아니라 "wide range"(100-240V 단일 연속 범위) 단일 대역으로 명시된다 — OM "Rated mains voltage 100-240V, 50-60Hz" 및 A&E "mains range of 100 to 240 V AC". LA12X/LA7.16의 "100-240(+/-10%)" 단일 연속 범위 방식과 유사한 구조.

**[2] Nominal_Current**: OM/SPS "Rated mains current 5.7A - 2.4A" — wide range 단일 사양이므로 고전압/저전압 개별 값이 아니라 "고전류 한계(저전압대에서 5.7A) - 저전류 한계(고전압대에서 2.4A)"로 해석하여 Nominal_Current_100_120V=5.7A, Nominal_Current_200_240V=2.4A에 매핑했다 — D25/D40/D80/D90의 "고전압대/저전압대 개별 정격 전류" 구조와 근본적으로 다른 wide-range 단일 사양이라는 점을 명시한다(임의 재해석이 아니라 원문의 유일한 두 수치를 기존 Key 구조에 최대한 근접 매핑한 것).

**Power_Factor**: 5D 소스 어디에도 명시적 역률 수치가 없어 null 처리했다. PFC 기능 자체는 탑재되어 있음(PSU_Type 참조).

**[4] PSU_Mains_Rated_Power**: OM Chapter 5.2.1 "Mains connection" 본문에 "Mains voltage range: 100 to 240 VAC, ~50/60 Hz, 450 W"로 명시적 단일 수치가 확인된다 — D25/D40과 달리 이 항목은 스펙 표가 아닌 서술 챕터에 명확한 단일 값으로 제공되어 PDF 레이아웃 뒤섞임 문제가 없었다.

**Mains_Current_Limiter_Range**: 5D 소스(축약판 스타트업 매뉴얼)에는 Mains Current Limiter 챕터 자체가 없다 — 완전판 Reference Manual 미제공으로 인한 미확인(null)이며, 기능 자체의 비존재를 의미하지 않는다(위 "소스 제약 사항" 참조).

"Dante input"이 D25/D40/D90의 "Digital/Milan input" 항목에 대응하는 것으로 매핑했다 — 물리적으로 동일한 개념(디지털 네트워크 입력의 절대 노이즈 전압)이나 프로토콜 명칭이 다름을 명시한다. 데이터시트 서두의 "dynamic range of 111 dB (unweighted)"는 이 표의 "Dante input, unweighted: 111 dB" 행과 일치한다(SPS/OM 교차 확인).

**측정 조건**: OM/SPS "Latency analog/Dante input (48 kHz, incl. Dante latency) 1.1/3 ms" — 아날로그 입력 1.1 ms, Dante 입력 3 ms(Dante 자체 지연 포함)로 서로 다르다. Latency_Low_Mode는 별도 개념이 확인되지 않아 비적용(null). **[신규] Web_Remote_Interface (10D 파싱 중 신설, 양방향 동기화로 null 반영)**: 10D(d&b audiotechnik 차세대 설치용 라인) 파싱 중 발견된 신규 Key — 별도 소프트웨어(R1/ArrayCalc) 없이 표준 웹 브라우저로 앰프에 직접 접속하는 브라우저 기반 내장 웹 인터페이스 개념. 5D의 기존 소스에는 이 개념에 대응하는 서술이 확인되지 않아 null(미확인)로 소급 반영한다.

## 버전 변경 이력

v1.0: Phase 3-2, d&b Dante 설치용 라인 최초 투입(D80/D90/D25/D40의 Milan/터치스크린 라인과 완전히 다른 아키텍처). SPS(Datasheet 1.1, 4p), OM(Start-up manual 1.3, 23p — 완전판 Reference Manual 아님, "소스 제약 사항" 참조) 2개 소스를 교차 검증하여 마스터 스키마 최초 구축. D80_v2.0.md를 구조적 뼈대로 복제했으나 대부분의 Value/개념이 새로 작성됨(Euroblock 커넥터 전용, LED-only UI, 9.5" 하프랙, 48kHz DSP, 설치용 저출력 75W급). 신규 스키마 확장 2건: (1) input_dante 신규 섹션(7개 Key: Dante_RX_Channels, Dante_Sampling_Rates, Dante_SRC, Dante_Redundancy, Dante_Network_Flows, Dante_Latency, Dante_IP_Architecture) — Milan-AVB 기준으로 설계된 input_avb 섹션과 별도로 Dante 프로토콜 전용 섹션을 신설. (2) connectivity 섹션에 Fault_Contact_Type 신규 Key(FAULT 릴레이 접점, NO/C/NC). 두 확장 모두 기존 11개 파일(D80, D90, D25, D40, LA12X, LA7.16, LA1.16i, LA2Xi, LA4X, LA7.16i)에 null로 소급 반영 필요(세션 후속 조치 대상). Bridging_Support는 OM에 "Bridge mode is not applicable."이라는 명시적 확정 문구가 있어 No로 처리(LA4X/LA7.16i와 동일 수준의 확정적 비존재). GPIO_Count=4로 최초로 실값이 채워진 사례(D80/D90은 0으로 확정적 비존재). 축약판 소스(완전판 Reference Manual 미제공) 한계로 다수의 세부 기능 Key(MCL, EcoMode, AmpPresets, LoadMatch, Speaker_Protection 등)가 미확인(null) 처리되었으며, 이는 기능 부재가 아닌 소스 제약임을 각 항목에 명시했다.

v1.1: 신규 제품 10D 파싱 중 control_monitoring 섹션에서 신설된 Web_Remote_Interface Key를 양방향 동기화 규칙에 따라 본 파일에 null로 소급 반영. 5D 자체 소스(OM/SPS)에는 이 개념에 대응하는 서술이 없어 미확인(null)로 처리했다. control_monitoring 섹션의 Signal_Path_Management 행 다음에 신규 행 추가, 해당 섹션 주석에 신설 근거 각주 추가. 그 외 기존 Key/Value/각주/구조는 v1.0과 완전히 동일하며 실제 스펙 데이터 변경은 없음(순수 스키마 동기화 목적의 Minor 버전).
