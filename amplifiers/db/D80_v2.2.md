d&b audiotechnik은 SKILL v1.8 기준 Phase 3-2로 최초 투입되는 브랜드로, L-Acoustics 전용 "OM 최우선" 규칙이 아닌 기본값("가장 상세하고 개정 이력 있는 문서 우선")을 적용한다. 본 소스 집합에서는 OM(1.14판, 개정 이력 명시)이 유일하게 상세 스펙 표와 챕터별 세부 설명을 모두 포함하는 문서이므로, 실질적으로 OM이 최우선 출처가 된다. AE(txt, undated)와 웹 3계층은 보조 출처로 사용한다. 별도의 SPS(스펙시트 단독 PDF)는 이번 소스 집합에 포함되지 않았다.

본 파일은 SKILL v1.7의 "완벽한 양방향 스키마 동기화" 규칙에 따라 LA12X_v2.5.md, LA7.16_v1.3.md와 Key 목록을 100% 일치시킨 최초 개정본이다(v1.0). D80 파싱 중 발견된 신규 속성은 [신규] 표시로, LA12X/LA7.16에만 있던 기존 속성 중 D80에 역주입된 Key는 [양방향 동기화] 표시로 구분했다. v1.1에서는 동일 브랜드 신모델 D90(Phase 3-1) 투입에 따라 신규 섹션 `d90_specific`(9개 Key)이 전량 `null`로 추가되었다. v1.2에서는 사용자가 DSP_Sampling_Rate/Latency 값의 성립 조건에 대한 근거 부족을 지적하여, OM 본문(p.34 "System clocking")을 재검토해 dsp/latency 섹션 각주를 보강했다 — 기존 Value/Key는 변경하지 않았다.

**Phase 4 동기화 안내(v1.3)**: Upload 폴더 4개 신규 제품(LA1.16i, LA2Xi, LA4X, LA7.16i, L-Acoustics 브랜드)의 파싱 과정에서 발견된 15개 신규 Key(브릿지 모드 관련 10개, AES67_Support, Service_Port_Type, Speaker_Terminal_Block_SE_BTL_Wiring, PBTL_Activation_Method)를 아래 각 섹션에 `null`로 신설 추가했다. D80의 기존 소스(OM 1.14, AE, 웹)에는 이들 개념에 대응하는 서술이 전혀 확인되지 않으며, 원본 문서를 재조회하지 않고 기존 파싱 결과물 텍스트만을 근거로 판단했으므로(사용자 승인에 따른 진행 방식) "비적용"으로 단정하지 않고 **미확인(null)**으로 보수적으로 처리했다 — D80은 애초에 L-Acoustics의 Milan-AVB 계열과 무관한 CAN-Bus/OCA 기반 제어 아키텍처를 사용하는 타 브랜드 제품이라는 점도 고려하여, 추후 D80 원본 OM 재검토 시 갱신 필요. 기존 9개 섹션 및 d80_specific/d90_specific의 Key/Value/각주는 일절 변경하지 않았다.

**구조 개편 안내(v2.0)**: d&b 제품군이 D80/D90에 D25가 추가되며 3개로 늘어남에 따라, 사용자 요청으로 "d80_specific"/"d90_specific"처럼 특정 모델명을 딴 그랩백(grab-bag) 섹션을 전부 폐지하고 그 안의 Key들을 주제(스펙 영역) 기준 기존 섹션으로 재분배했다. **d80_specific(12개 Key)**: CMRR_Analog_Input -> input_analog, Digital_Input_Impedance -> input_digital, SN_Ratio_Analog_Input/SN_Ratio_Digital_Input -> audio_performance, System_Start_Up_Time/LoadMatch_Max_Cable_Length/Load_Monitoring_System_Check -> dsp, Storage_Temp_Range/Storage_Humidity -> operating_conditions, Signal_Path_Management -> control_monitoring, Output_Mode_Selectable -> connectivity, Mains_Current_Limiter_Range -> power_supply. **d90_specific(9개 Key)**: Milan_Device_Type은 input_avb의 명명 규칙에 맞춰 AVB_Device_Type으로 개명 후 재배치, Milan_Input_Channel_Streams/Milan_Redundancy/Milan_Routable_Inputs는 input_avb의 기존 AVB_Channels/AVB_Stream_Count/Redundancy_Type과 중복되어 흡수(신규 Key 생성 없음), Time_To_Tone/Energy_Saving_Detail/AmpPresets_Detail -> dsp, Output_Noise_Dynamic_Range_Detail -> audio_performance, Network_IP_Default -> 신설된 **network** 섹션(주제 기반 신규 섹션, 향후 Dante 라인 제품의 네트워크 설정 Key도 여기 수용 예정). 아울러 사용자 피드백에 따라 BTL/PBTL 전력 Key 10개(부하 임피던스별 세분화)를 4개(Rated/Peak_Power_Per_Ch_BTL, Rated/Peak_Power_Per_Ch_PBTL)로 통합하여 null 나열로 인한 가독성 저하를 해소했다(실값 있는 제품은 Value 셀에 임피던스 조건을 슬래시로 병기). Key 이름/Value/Unit과 각주 내용 자체는 변경하지 않고 위치만 이동했으며(BTL/PBTL과 Milan 통합 부분만 예외), 각주 번호는 이동 대상 섹션 내에서 기존 번호와 충돌하지 않도록 재부여했다. 섹션 계층 구조 자체가 바뀌는 변경이므로 SKILL v1.8 버저닝 규칙에 따라 Major 버전(v2.0)으로 상향했다.

### 주석 및 출처 (Notes & Sources)

**출처**: OM p.9 "Technical specifications" (최우선) / AE (txt) / WEB-F

**측정 조건**: 전 채널 구동, THD+N < 0.5% 기준. Amplification_Gain은 "Linear mode @ 0 dB" 조건(OM p.9). Output_Delay_Range는 채널별 독립 적용, 최대 10초(= 3440 m / 11286 ft, OM p.10 "Delay", AE 일치).

**[1] Rated_Power_Per_Ch 표 구조 차이 (d&b 브랜드 고유 표기)**: D80의 OM/AE/WEB-F는 LA12X/LA7.16처럼 부하 임피던스(Ohm)별로 단일 값을 명시하는 대신, Crest Factor(CF) 조건별로 4/8 Ohm 두 부하의 값을 동시에 병기하는 방식을 사용한다("CF = 6 dB @ 4/8 ohms: 4 x 2600/2000 W", "CF = 12 dB @ 4/8 ohms: 4 x 4000/2000 W"). 기존 LA12X/LA7.16 스키마의 `Rated_Power_Per_Ch_*_Ohms`는 CEA-2006/490A 등 표준 RMS 조건 단일값을 의미하는 Key였으나, D80 소스에는 이런 표준 규격명 대신 CF(Crest Factor) dB값으로 조건이 구분되어 있어 두 개념이 완전히 대응되지 않는다. 무결성 원칙(임의 추론/수식어 첨가 금지)에 따라 값을 자의적으로 재분류하지 않고, `Rated_Power_Per_Ch_4_Ohms`/`_8_Ohms`에는 "CF 6 dB 값 / CF 12 dB 값" 형식으로 두 조건 값을 슬래시로 병기했다(4 Ohms: 2600/4000 W, 8 Ohms: 2000/2000 W). Rated_Power_Per_Ch_2.7_Ohms와 16_Ohms 조건은 D80 소스 어디에도 존재하지 않아 비적용(null) 처리했다.

**CF(Crest Factor) 조건별 상세값** (OM p.9, AE 일치): CF = 6 dB @ 4 ohms = 2600 W, CF = 6 dB @ 8 ohms = 2000 W, CF = 12 dB @ 4 ohms = 4000 W, CF = 12 dB @ 8 ohms = 2000 W (전 채널 구동, THD+N < 0.5%). `Peak_Power_Per_Ch_4/8/16_Ohms`는 D80 소스에 해당 명칭의 별도 항목이 없어 null로 유지한다.

**[2] Total_Power_Capability**: D80의 OM/AE/WEB-F 어디에도 전 채널 동시 구동 시 총 출력 용량을 나타내는 단일 수치가 명시되어 있지 않다. OM p.10 "Max. power consumption (short term RMS) 7000 W"는 메인즈로부터 끌어오는 단기 입력 전력이며(power_supply 섹션 PSU_Mains_Rated_Power와 별개 개념), 채널당 정격의 단순 합산(4 x 4000 W = 16000 W)은 소스에 명시된 값이 아니므로 임의 계산하지 않는다(총량/정격 Key 오염 금지 원칙). 따라서 null 처리.

**[신규] Max_Output_Current_Peak**: D25 파싱 중 신설된 Key(D25는 OM/SPS에 "Maximum output voltage/current 120 Vpeak/35 Apeak"로 최대 출력 전류가 명시됨). D80의 기존 소스(OM 1.14, AE, 웹)에는 이에 대응하는 전류 값이 확인되지 않아 null(미확인)로 소급 반영한다.

**[26] Bridging_Support 및 BTL/PBTL 관련 Key (Phase 4 동기화)**: L-Acoustics "i" 설치전용 라인(LA1.16i, LA2Xi) 파싱 중 발견된 브릿지 모드(SE/BTL/PBTL) 관련 속성군. D80의 기존 소스(OM 1.14, AE, 웹)에는 브릿지/BTL/PBTL 관련 서술이 확인되지 않는다 — 원본 문서를 재조회하지 않은 보수적 처리 방식에 따라 null(미확인)로 유지한다. D80은 Dual Channel/Mix TOP-SUB/2-Way Active 등 채널쌍 단위 출력 모드 선택 기능(connectivity 섹션 Output_Mode_Selectable 참조)을 갖고 있으나, 이는 L-Acoustics의 SE/BTL/PBTL 브릿지 개념과 명명 및 작동 원리가 다른 것으로 판단된다(브릿지는 두 채널의 앰프 출력을 합산해 더 높은 전압/전력을 만드는 방식인 반면, D80의 출력 모드는 채널 라우팅/필터링 방식 선택에 가깝다) — 다만 이 판단도 D80 OM 재검토 없이는 확정할 수 없어 null로 유지한다.

**[26a][구조 개편 v2.0] BTL/PBTL 전력 Key 통폐합 (가독성 개선)**: v1.3까지는 부하 임피던스(8/16 Ohm, PBTL은 4/8/16 Ohm)별로 Rated/Peak를 전부 분리한 10개 Key(Rated/Peak_Power_Per_Ch_BTL_8/16_Ohms, Rated/Peak_Power_Per_Ch_PBTL_4/8/16_Ohms)였다. 브릿지 모드를 지원하지 않거나 미확인인 제품(D80/D90/LA12X/LA7.16/LA4X/LA7.16i)에서 이 10개 Key가 전부 null로 나열되어 가독성이 크게 떨어진다는 사용자 피드백에 따라, 4개 Key(Rated_Power_Per_Ch_BTL, Peak_Power_Per_Ch_BTL, Rated_Power_Per_Ch_PBTL, Peak_Power_Per_Ch_PBTL)로 통합했다. 실값이 있는 제품(LA1.16i, LA2Xi)은 임피던스 조건을 Value 셀 안에 "값1 @ n Ohms / 값2 @ m Ohms" 형식으로 병기한다(Rated_Power_Per_Ch_4_Ohms의 CF6dB/CF12dB 슬래시 병기 관례와 동일한 방식). D80은 브릿지 관련 서술 자체가 없어 4개 Key 모두 null 유지.

### 주석 및 출처 (Notes & Sources)

**출처**: OM p.9 "Technical specifications" (최우선) / AE (txt)

**측정 조건**: Frequency_Response는 -1 dB 허용오차 기준. THD_N_4ohm은 "20 Hz - 20 kHz, 600 W @ 4 ohms" 조건. Damping_Factor는 "20 Hz - 200 Hz into 4 ohms" 조건. SN_Ratio는 "unweighted, RMS" 조건.

**[8a][구조 개편 v2.0] SN_Ratio_Analog_Input/SN_Ratio_Digital_Input**: v1.x까지 "d80_specific" 섹션에 있던 Key를 audio_performance로 재배치(Key/Value/Unit 및 근거는 변경 없음). Output_Dynamic_Range/Noise_Level과는 물리량 기준이 달라 별도 Key로 유지한다.

**[8b][구조 개편 v2.0] Output_Noise_Dynamic_Range_Detail**: v1.x까지 "d90_specific" 섹션에 있던 Key를 audio_performance로 재배치(Output_Dynamic_Range/Noise_Level과 인접 배치, 근거는 변경 없음). D80은 이 상세 절대 노이즈 전압 기반 데이터 구조에 대응하는 서술이 확인되지 않아 null 유지.

### 주석 및 출처 (Notes & Sources)

**출처**: OM p.9-10 "Technical specifications", p.14 "6.2.2 Audio INPUT and LINK connectors", p.33-34 "11.2.1 Input mode > System clocking", p.69 "12.8.2 LoadMatch" (최우선) / AE (txt) / WEB-F

**측정 조건**: 없음(사양값). DSP_Sampling_Rate는 "96 kHz/27 Bit ADC/24 Bit DAC" 조건(OM/AE/WEB-F 일치).

**[신규][9a] DSP_Sampling_Rate 값의 의미 명확화 (내부 고정값 vs 입력 허용 범위 구분, 사용자 검토 반영)**: 사용자 검토 결과, "96 kHz"라는 단일 수치가 "96 kHz까지 지원"(가변 상한)인지 "96 kHz 고정 동작"인지 원문 확인이 필요하다는 지적이 있어 OM p.34 "System clocking" 절 본문을 재검토했다. 확인 결과 D80의 DSP 내부 처리 클록은 항상 96 kHz로 고정이며, 가변 상한이 아니다 — OM p.34 원문: "The clock of the digital audio system is derived from an internal crystal oscillator with a sampling rate of 96 kHz... It is also possible to use a signal with a sampling rate of 48 kHz... In this case, the system detects the sampling rate and automatically doubles it using a synchronous sampling rate doubler to achieve the required 96 kHz." 즉 48 kHz 입력이 들어오면 자동으로 2배 업샘플링되어 내부적으로는 항상 96 kHz로 처리된다. Value를 "96 (internal, fixed)"로 명확화했다(수치 자체는 변경 없음, 의미 명시만 보강).

**SRC(Sample Rate Converter) 동작 조건과 레이턴시 영향 (OM p.34 "SRC" 절)**: D80은 기본적으로(by default) SRC를 사용하지 않는다 — "To keep latency as short as possible, the system does not utilize (asynchronous) Sample Rate Converters (SRC) by default." 디지털 입력이 48 또는 96 kHz(내부 클록과 정수배 관계)가 아닌 다른 샘플레이트(예: 44.1, 88.2 kHz)로 들어오거나 두 디지털 입력이 서로 다른 소스에서 비동기로 들어올 경우에 한해 SRC를 수동으로 활성화해야 하며, 이 경우 "slightly increased basic latency ≤ 1 ms"(기본 레이턴시에 최대 1 ms 추가)가 발생한다(OM p.34). 즉 D80의 Latency_Standard(0.3 ms, 아래 latency 섹션 참조)는 "SRC 비활성 상태, 48/96 kHz 입력"이라는 조건에서만 성립하는 값이며, 무조건적인 고정값이 아니다.

**LA12X/LA7.16과의 구조적 차이 (양방향 참고, 근거 유무 명시)**: L-Acoustics LA12X/LA7.16은 반대로 모든 AES/EBU 입력에 SRC가 항상(상시) 내장된 하드웨어로 존재하며(LA12X OM p.15 "Each AES/EBU input port is equipped with an SRC... The SRC converts the formats to the 24 bits / 96 kHz internal format... and provides constant propagation delay regardless of the input sampling frequency"), 조건부로 켜고 끄는 개념이 아니다. 그 결과 LA12X/LA7.16의 Latency 스펙 표에는 "independent from input Fs"(입력 샘플레이트와 무관하게 일정)라는 문구가 명시적으로 붙어 있다(LA12X_v2.9.md, LA7.16_v1.7.md latency 섹션 참조). D80은 이와 달리 SRC가 기본 비활성 + 조건부 활성화 구조이며 활성화 시 레이턴시가 변동한다는 점에서 근본적으로 다른 아키텍처다. D80의 OM에는 "SRC가 상시 내장되어 있다"는 서술이 없고 오히려 "SRC를 기본적으로 사용하지 않는다"고 명시되어 있다는 점에서, LA12X/LA7.16과 정반대의 설계 철학을 보여준다.

**DSP_Processor, Routing_Matrix, Per_Channel_DSP_Tools**: D80 소스 어디에도 DSP 프로세서 칩셋명이나 라우팅 매트릭스 규격이 확인되지 않아 null 처리했다. Per_Channel_DSP_Tools는 LA7.16의 "어레이/음향 보정용 필터 도구"와 성격이 다른 범주(신호 감시/전환/보호 로직)이므로 아래 [신규] Key로 별도 신설했다.

**[9] EQ_Filters_Per_Output 값**: OM p.10 "two user definable 16-band equalizers... Filter types: PEQ/Notch/HiShlv/LoShlv/Asym".

**[10] Amplifier_Power_Management 매핑**: LA7.16의 L-SMART에 대응하는 D80 기능은 "Mains Current Limitation (MCL): 95 to 50% of 16/30 A"(OM p.9). 작동 원리가 다를 수 있어 임의로 동일시하지 않고 원문 명칭(MCL)을 채택했다.

**[11] Speaker_Protection 매핑**: LA12X/LA7.16의 L-DRIVE에 정확히 대응하는 기술이 D80에 없어, LoadMatch/ArrayProcessing을 병기했다(성격은 다름 — 보호가 아닌 보정 기능).

**[11a][구조 개편 v2.0] LoadMatch_Max_Cable_Length/Load_Monitoring_System_Check/System_Start_Up_Time**: v1.x까지 "d80_specific" 섹션에 있던 3개 Key를 dsp로 재배치(Key/Value/Unit 및 근거는 변경 없음). LoadMatch_Max_Cable_Length는 "up to 70 m", 20 kHz 대역폭까지 보상하는 조건(OM p.69).

**[11b][구조 개편 v2.0] Time_To_Tone/Energy_Saving_Detail/AmpPresets_Detail**: v1.x까지 "d90_specific" 섹션에 있던 3개 Key를 dsp로 재배치(System_Start_Up_Time과 같은 전원/프리셋 관련 성격이므로 인접 배치, Key/Value 및 근거는 변경 없음). D80은 이 3개 개념에 대응하는 서술이 확인되지 않아 null 유지.

### 주석 및 출처 (Notes & Sources)

**출처**: OM p.9-10 "Technical specifications", p.34 "System clocking > SRC" (최우선) / AE (txt)

**측정 조건**: OM p.9 "Latency analog and digital inputs 0.3 msec", OM p.10 "Latency analog input 0.3 msec / Latency digital input (AES) 0.3 msec, 48 kHz/96 kHz".

**[12] Latency_Standard/Low_Mode 구조 차이**: D80 소스에는 단일 지연시간(0.3 ms)만 명시되어 있고 별도의 "Low latency 모드" 개념이 확인되지 않아 null 처리했다. AVB_Max_Network_Latency는 D80이 AVB/Milan을 사용하지 않으므로 비적용(null).

**[신규][12a] Latency_Standard(0.3 ms) 성립 조건 명확화 (사용자 검토 반영)**: OM p.9-10 스펙 표의 "0.3 msec" 값은 "48 kHz/96 kHz" 조건이 병기되어 있으나, 이 조건의 정확한 의미는 스펙 표만으로는 불명확했다. OM p.34 "System clocking" 절 본문을 대조한 결과, 이 조건은 "SRC가 비활성 상태이고 입력 신호가 48 kHz 또는 96 kHz인 경우"를 의미하는 것으로 확인된다. D80은 SRC를 기본적으로 사용하지 않으며, 44.1/88.2 kHz 등 비표준 샘플레이트 입력이나 비동기 다중 디지털 소스 입력 시에는 SRC를 수동 활성화해야 하고, 이 경우 "slightly increased basic latency ≤ 1 ms"(OM p.34)로 레이턴시가 증가한다 — 즉 실제 레이턴시는 조건에 따라 0.3 ms ~ 최대 약 1.3 ms 범위로 변동할 수 있다. Value에 조건을 병기하여 이 수치가 무조건적 고정값이 아님을 명확히 했다(원래 수치 0.3 ms 자체는 변경 없음).

**LA12X/LA7.16과의 서술 유무 차이 (양방향 참고)**: LA12X/LA7.16의 OM은 Latency 스펙 표에 "independent from input Fs"라는 문구가 명시적으로 붙어 있어(LA12X OM p.86, LA7.16 OM p.94) 조건에 관계없이 고정값임이 원문에서 확정된다. 반면 D80의 OM에는 이런 확정 문구가 없고, 오히려 SRC 조건에 따라 레이턴시가 달라질 수 있다는 서술(OM p.34)이 존재한다 — L-Acoustics와 d&b D80 사이의 실제 아키텍처 차이(SRC 상시 내장 vs 조건부 활성화)에서 비롯된 것으로 판단되며, D80 소스에 "고정값"이라는 확정적 서술이 없다는 사실 자체를 이 각주에 명기한다.

## 버전 변경 이력

v1.0: Phase 3-2 신규 브랜드(d&b audiotechnik) 최초 파싱. OM(93p), AE(txt), 웹 3계층 총 5개 소스를 교차 검증하여 마스터 스키마 최초 구축. LA12X/LA7.16 대비 12개 신규 속성(d80_specific 섹션)을 발견하여 소급 반영함.

v1.0 to v1.1: (Phase 3-1 양방향 동기화) 동일 브랜드 신모델 D90 파싱 중 발견된 신규 섹션 `d90_specific`(9개 Key)을 전량 `null`로 신설 추가.

v1.1 to v1.2: (사용자 검토 반영) dsp 섹션의 DSP_Sampling_Rate와 latency 섹션의 Latency_Standard 값이 성립하는 정확한 조건에 대한 근거가 부족하다는 지적에 따라, OM p.34 "System clocking" 절 본문을 재검토하여 두 Key의 각주를 대폭 보강했다. 핵심 확인 사항: (1) DSP_Sampling_Rate(96 kHz)는 내부 처리 고정값이며 가변 상한이 아니다 — 48 kHz 입력은 자동으로 2배 업샘플링되어 항상 96 kHz로 처리된다. (2) Latency_Standard(0.3 ms)는 "SRC 비활성, 입력 48/96 kHz" 조건에서만 성립하며, D80은 SRC를 기본적으로 사용하지 않다가 44.1/88.2 kHz 등 비표준 입력 시에만 수동 활성화하는 구조로, 이 경우 레이턴시가 최대 1 ms까지 추가된다(OM p.34). (3) 이 조건부 구조는 L-Acoustics LA12X/LA7.16의 "SRC 상시 내장 + independent from input Fs 명시" 구조와 정반대이며, 이 아키텍처 차이를 양방향 동기화 관점에서 각 파일에 명기했다. Value 자체(96, 0.3)는 변경하지 않고 성립 조건만 병기했다.

v1.2 to v1.3: (Phase 4 양방향 동기화) Upload 폴더 4개 신규 제품(LA1.16i, LA2Xi, LA4X, LA7.16i) 파싱 중 발견된 신규 15개 Key(amplification 10개: Bridging_Support + BTL 전력 4개 + PBTL 전력 6개, input_avb 1개: AES67_Support, connectivity 3개: Service_Port_Type, Speaker_Terminal_Block_SE_BTL_Wiring, PBTL_Activation_Method)를 각 해당 섹션에 `null`로 신설 추가. D80은 L-Acoustics와 다른 브랜드(d&b audiotechnik)이며 기존 소스에는 이들 개념에 대응하는 서술이 없어 "미확인(null)"으로 보수적 처리(원본 문서 재조회 없이 기존 파싱 결과 텍스트 기반 사용자 승인 진행). 기존 9개 섹션 및 d80_specific/d90_specific의 Key/Value/각주는 일절 변경하지 않았다.

v1.3 to v2.0: (구조 개편, Major) d&b 제품군이 D25 투입으로 3개(D80/D90/D25)로 늘어남에 따라, 사용자 요청으로 "d80_specific"과 "d90_specific" 두 그랩백 섹션을 모두 폐지했다. d80_specific의 12개 Key는 CMRR_Analog_Input -> input_analog, Digital_Input_Impedance -> input_digital, SN_Ratio_Analog_Input/SN_Ratio_Digital_Input -> audio_performance, System_Start_Up_Time/LoadMatch_Max_Cable_Length/Load_Monitoring_System_Check -> dsp, Storage_Temp_Range/Storage_Humidity -> operating_conditions, Signal_Path_Management -> control_monitoring, Output_Mode_Selectable -> connectivity, Mains_Current_Limiter_Range -> power_supply로 재배치(Key/Value/Unit/각주 내용 유지, 위치만 이동). d90_specific의 9개 Key는 Milan_Device_Type -> input_avb(AVB_Device_Type으로 개명), Milan_Input_Channel_Streams/Milan_Redundancy/Milan_Routable_Inputs -> input_avb의 기존 AVB_Channels/AVB_Stream_Count/Redundancy_Type에 흡수(중복 제거, 신규 Key 미생성), Time_To_Tone/Energy_Saving_Detail/AmpPresets_Detail -> dsp, Output_Noise_Dynamic_Range_Detail -> audio_performance, Network_IP_Default -> 신설된 network 섹션으로 재배치. 아울러 사용자 피드백에 따라 BTL/PBTL 전력 Key 10개(부하 임피던스별 세분화)를 4개(Rated/Peak_Power_Per_Ch_BTL, Rated/Peak_Power_Per_Ch_PBTL)로 통합했다(D80은 전부 null이므로 값 손실 없음). 이동한 Key마다 원래 위치를 명시하는 각주를 신설했다. Null Report 총 건수는 60건에서 59건으로 변경(Milan 중복 3개 Key 흡수로 감소, BTL/PBTL 통합으로 6개 감소, AVB_Device_Type 신규 1개 추가 — 순증감 상쇄). 섹션 계층 구조 변경이므로 Major 버전(v2.0)으로 상향.

v2.0 to v2.1: (양방향 스키마 동기화, Minor — null-only 추가, D80 실데이터 변경 없음) d&b 신규 제품 5D(Dante 기반 설치용 라인, Phase 5) 파싱 중 발견된 신규 스키마 확장 2건을 D80에도 동일하게 반영했다. (1) 신규 섹션 `input_dante`(Dante_RX_Channels, Dante_Sampling_Rates, Dante_SRC, Dante_Redundancy, Dante_Network_Flows, Dante_Latency, Dante_IP_Architecture 7개 Key)를 input_avb 섹션 직후, network 섹션 직전에 신설하고 전량 `null`로 추가했다 — D80은 AES/EBU(AES3) + CAN-Bus/OCA 기반 제품으로 Dante를 사용하지 않으므로 섹션 전체가 비적용이다. (2) connectivity 섹션에 `Fault_Contact_Type`(FAULT 릴레이 접점 NO/C/NC) Key를 PBTL_Activation_Method 다음 행에 `null`로 신설 추가했다 — 하드웨어적으로 존재 가능성을 배제할 근거가 없어 "미확인"으로 분류(비적용 아님). 두 확장 모두 5D의 실제 소스(OM Chapter 5.2.3/5.2.6)에서 확인된 개념이며 기존 스키마 Key로 흡수 불가능해 신규 신설되었다(양방향 동기화 규칙에 따라 기존 11개 파일 전체에 동일 반영). 아울러 이번 작업의 grep 검증(`| null` 실카운트) 과정에서 v1.3~v2.0 동안 Null Report 본문 나열에서 누락되어 있던 기존 Key `Redundancy_Type`(input_avb, 비적용) 1건을 발견하여 비적용 목록에 보정 추가했다 — 테이블상 Value는 이미 v1.0부터 null이었고 실제 스키마에는 변경이 없으며, Null Report의 서술 누락만 바로잡은 것이다. 이번 버전에서 Key/Value가 실제로 변경된 대상은 D80 소스 자체에는 없으며(전량 null 추가), Null Report 총계만 60건 -> 69건으로 갱신되었다(신규 8건 + 보정 1건). 섹션 내부 Key 추가와 신규 섹션 신설(구조 확장이나 기존 섹션 순서/체계 자체의 재편은 아님)이므로 Minor 버전(v2.1)으로 상향.

v2.1 to v2.2: (양방향 스키마 동기화, Minor — null-only 추가, D80 실데이터 변경 없음) d&b 신규 제품 10D(차세대 설치용 라인, Phase 6) 파싱 중 발견된 신규 Key `Web_Remote_Interface`(control_monitoring, Signal_Path_Management 다음 행)를 null로 소급 반영했다. 별도 소프트웨어(R1/ArrayCalc) 없이 표준 웹 브라우저로 앰프에 직접 접속하는 브라우저 기반 내장 웹 인터페이스 개념으로, D80은 TFT 터치스크린(Front_Panel) 기반 로컬 UI를 갖추고 있으나 이는 브라우저 기반 웹 인터페이스와는 다른 개념이며 D80의 기존 소스에는 이에 대응하는 서술이 확인되지 않아 null(미확인)로 처리했다. 양방향 동기화 규칙에 따라 기존 12개 파일 전체에 동일 반영. Null Report 총계는 69건 -> 70건으로 갱신되었다(신규 1건). 섹션 내부 Key 1건 추가에 불과하므로 Minor 버전(v2.2)으로 상향.
