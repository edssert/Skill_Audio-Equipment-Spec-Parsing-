# amplifiers 카테고리 폴더 지침

이 폴더는 앰프/컨트롤러 제품군을 브랜드별 하위 폴더로 관리한다. 브랜드 폴더명: L-Acoustics -> `LA`, d&b audiotechnik -> `db`.

각 브랜드 폴더(`LA/`, `db/`)는 아래 내부 구조를 동일하게 따른다:

- `{제품명}_v{버전}.md` — 해당 제품의 최신 마스터 스키마 파일 (브랜드 폴더 루트에 위치)
- `Original_PDFs/{제품명}/` — 원본 문서(OM/SPS/AE) 보존. 파일명 규칙: `{제품명}_{문서종류}_EN_{버전}.{확장자}` (문서종류: OM=매뉴얼, SPS=스펙시트, AE=제품소개). 버전은 문서 헤더/푸터 표기값을 그대로 사용, 못 찾으면 리네임하지 않음. AE 문서는 버전 정보가 없는 게 정상. 원본은 절대 수정하지 않는다.
- `Raw_Web_Data/{제품명}_Raw_Web_Data.md` — 웹사이트 계층별 텍스트(리스트뷰/오버뷰/Full Spec) 및 비-PDF 원문(txt A&E 등)을 가공 없이 병합 보존.
- `Archive/{제품명}/` — 해당 제품의 과거 버전 마스터 스키마(`{제품명}_v{버전}.md`) 보관. 최신 버전은 브랜드 폴더 루트의 `{제품명}_v*.md` 참고, 여기는 이전 버전 이력용. 새 버전이 갱신되면 이전 파일을 이 폴더로 옮겨 보관.

새 브랜드가 투입되면 이 규칙과 동일한 내부 구조(제품 스키마 루트 + Original_PDFs + Raw_Web_Data + Archive)를 가진 신규 하위 폴더를 `amplifiers/` 아래에 생성한다.

향후 스피커(speakers), 앰프렉(amp_racks) 등 신규 제품 카테고리 추가 시에도 이 폴더와 동일한 브랜드 하위 폴더 패턴(카테고리/브랜드/...)을 루트에 병렬로 적용한다.
