**스켈레톤 재사용 근거 (SKILL v1.11 "신규 제품 스켈레톤 재사용 규칙")**: 본 파일은 D90_v2.0.md의 Key 목록/섹션 구조를 뼈대로 복제한 뒤 서지컬 교체하여 작성되었다. D25는 D80보다 D90과 아키텍처가 훨씬 가깝다고 판단한 근거: (1) Milan™ Endstation 오디오 네트워크 사용(D80은 CAN-Bus/OCA만 사용, Milan 없음), (2) TFT 컬러 터치스크린(4.3"/480x272 pixel, D90과 완전히 동일 사양) + 디지털 로터리 인코더 UI(D80은 3.5"/320x240의 더 작은 화면), (3) 아날로그/디지털 입력이 물리적으로 분리된 전용 커넥터 구조(D80은 4슬롯 중 2개 겸용 전환식), (4) LoadMatch/ArrayProcessing/System check/Load monitoring/AmpPresets/EcoMode/Fallback/Override/AutoStandby/AutoWakeup 등 상세 기능 세트가 D90과 거의 1:1 대응. D25 자체 소스에서 확인되는 실제 값 차이(전력 정격, 무게, CMRR, 팬 개수 등)만 D90 값 위에 정확히 교체했다.

이번 소스 집합은 D25 데이터시트(SPS, 4p — 마지막 페이지에 "Architectural specifications" 절이 그대로 포함되어 있어 별도 AE 문서 없이 이 SPS가 AE 역할까지 겸함)와 Reference Manual(OM, 71p) 2개 문서로 구성된다. 웹 3계층 텍스트는 이번 라운드에 제공되지 않아 Raw_Web_Data는 생성하지 않았다. 브랜드 기본 우선순위("가장 상세하고 개정 이력 있는 문서 우선", D80에서 확정)에 따라 OM(Reference Manual, 챕터별 상세 서술 포함)을 최우선 출처로 삼고 SPS를 병기한다 — 다만 두 문서의 스펙 표 수치는 실질적으로 100% 일치하여(SPS의 표가 OM Chapter 3 "Technical specifications"와 동일) 충돌 없이 상호 보완적으로 사용했다.

본 파일은 SKILL v1.11 기준 "완벽한 양방향 스키마 동기화" 및 "d80_specific/d90_specific 섹션 폐지, BTL/PBTL Key 통폐합" 구조 개편(D80_v2.0.md/D90_v2.0.md에서 이미 확정)이 반영된 최신 스키마 구조로 최초 작성되었다 — 즉 D25는 처음부터 개편된 구조(input_analog/input_digital/audio_performance/dsp/operating_conditions/control_monitoring/connectivity/power_supply에 재분배된 d&b 공통 Key, input_avb의 AVB_Device_Type, 신설된 network 섹션, 4개로 통합된 BTL/PBTL Key)로 작성되며, 구식 d80_specific/d90_specific 섹션 자체가 존재하지 않는다.

### 주석 및 출처 (Notes & Sources)

**출처**: OM p.8-10 "3 Technical specifications" (최우선) / SPS 2-4p

**측정 조건**: 전 채널 구동. Amplification_Gain은 "Linear mode @ 0 dB" 조건(OM/SPS 일치). Output_Delay_Range는 채널별 독립 적용, 최대 10초(OM/SPS 일치).

**[1] Rated_Power_Per_Ch 표 구조 (D90과 동일한 d&b 브랜드 고유 표기, D90과 완전히 동일한 세대)**: D25의 OM/SPS는 D90과 동일하게 두 조건의 채널당 출력을 명시한다 — (a) "Output power rating EIA-426B noise CF 12 dB": 8 Ω = 900 W, 4 Ω = 1800 W(순간/버스트성 정격), (b) "Sine wave 1 kHz, long term, +40°C": 4 Ω = 250 W(연속 정격, 열 제한 조건. 8 Ω 조건의 별도 long term 수치는 소스에 명시되지 않아 D90과 동일하게 4 Ω 조건에서만 확인된 값임을 각주로 명기). D25 OM은 이에 더해 EIA-426B의 CF 9 dB/6 dB 조건별 상세값도 제공한다(8/4 Ω 각각, OM p.10 "3.1" 절) — 이는 D80/D90에는 없던 추가 세분화이나 기존 스키마가 CF 12 dB 조건만 채택하므로 표에는 반영하지 않고 여기 각주로만 보존한다: EIA-426B CF 9 dB(8 Ω = 900 W, 4 Ω = 1300 W), CF 6 dB(8 Ω = 900 W, 4 Ω = 700 W). 1 kHz burst(20 ms/0 dB, 4 Ω) = 1050 W(230 ms 지속), 1 kHz sine(1채널 구동, 4 Ω) = 1800 W.

**[2] Total_Power_Capability**: D25의 OM/SPS 어디에도 전 채널 동시 구동 시 총 출력 용량을 나타내는 단일 수치가 명시되어 있지 않다. 채널당 정격의 단순 합산은 임의 계산 금지 원칙에 따라 채택하지 않는다. null 처리.

**[신규][3] Max_Output_Current_Peak**: OM/SPS "Maximum output voltage/current 120 Vpeak/35 Apeak". 기존 스키마(D80/D90/L-Acoustics 전 제품)에는 이 Key가 없어 신설했다 — Max_Output_Voltage_Peak Key와 짝을 이루는 전류 값이다. 양방향 동기화 규칙에 따라 기존 9개 파일에도 이 섹션에 `null`로 소급 반영이 필요하다(세션 후속 조치 대상).

**[4] Bridging_Support 및 BTL/PBTL 관련 Key**: D25의 OM(전문, "shall be" A&E 절 포함) 및 SPS 전체를 검토했으나 Bridge/BTL/PBTL/bridging 관련 용어나 절차가 전혀 확인되지 않는다. D25는 Dual Channel/Mix TOP-SUB/2-Way Active 등 채널쌍 단위 출력 모드(connectivity 섹션 Output_Mode_Selectable 참조)를 갖고 있으나, D80/D90과 마찬가지로 이는 L-Acoustics의 SE/BTL/PBTL 브릿지 개념과 명명 및 작동 원리가 다른 것으로 판단된다(D80_v2.0.md/D90_v2.0.md 동일 각주 참조). D25 자신의 원본 문서(OM/SPS)를 직접 전량 검토한 결과이므로 D80/D90보다 근거 수준이 높으나, "브릿지 기능 자체가 존재하지 않는다"는 확정적 서술(LA4X/LA7.16i처럼 "does not support"라는 명시적 부인 문구)은 없어 완전한 확정적 비존재(No)로 격상하지 않고 보수적으로 null 유지한다. Rated/Peak_Power_Per_Ch_BTL/PBTL 4개 Key(구조 개편 v2.0에서 10개->4개로 통합된 형태, D80_v2.0.md 참조)도 동일하게 null 유지.

### 주석 및 출처 (Notes & Sources)

**출처**: OM p.8 "3 Technical specifications", p.64-66 "10.1 Power supply" (최우선) / SPS 2p

**측정 조건**: Power_Consumption_Idle/Standby는 OM Chapter 3.1 "Current/power draw and thermal dissipation" 표의 230 VAC/50 Hz 대표 조건 기준(다른 전압 조건 - 208V/120V/100V - 은 OM p.11에 별도 표로 제공, 이 스키마는 D80/D90과 동일하게 230V 대표값만 채택). PSU_Type Value는 원문 그대로 채택 — D90과 완전히 동일한 표현.

**[1] Nominal_Current 값 산정**: OM/SPS "Rated mains current (High range) 13 A", "Rated mains current (Low range) 20 A". D80(고전압 16A/저전압 30A)·D90(16A/24A)과 모두 다른 값 — 제품 간 실제 사양 차이로 판단(충돌 아님).

**[2] Power_Consumption_Idle/Standby 값 산정 (OM Chapter 3.1 표, 산술 교차검증)**: OM/SPS 첫 페이지 요약란("Power consumption (typical values) / Standby / Idling / Peak output")은 PDF 레이아웃(다단 표) 추출 시 라벨과 수치가 뒤섞여 신뢰할 수 없었다(예: "21 W / 130 W / 2275 W"라는 3개 수치가 어느 상태에 대응하는지 확정 불가). 대신 OM Chapter 3.1의 "230 VAC / 50 Hz" 상태별 상세 표(Off/Standby/ReadyStandby/Eco/Idling, 각 상태의 Mains current[A]/Power factor/Input power[W] 3개 값)를 채택했다 — 이 표는 물리 법칙(전압×전류×역률=전력)으로 각 행의 정합성을 직접 검증할 수 있었다: Off(0.24A x 0.03 x 230V = 1.7W ≈ 1.5W), Standby(0.29A x 0.31 x 230V = 20.7W ≈ 20.8W 일치), ReadyStandby(0.43A x 0.54 x 230V = 53.4W ≈ 52.6W), Eco(0.83A x 0.54 x 230V = 103.1W = 103W 일치), Idling(0.75A x 0.74 x 230V = 127.7W ≈ 127W 일치) — 모든 행이 산술적으로 자체 정합되어 신뢰도가 높다고 판단, 이 표의 값을 채택했다. Power_Consumption_Idle=Idling 상태(127W), Power_Consumption_Standby=Standby 상태(20.8W)로 매핑(D80/D90과 동일한 매핑 규칙). ReadyStandby(52.6W)/Eco(103W) 상태는 D80/D90 스키마에 대응 Key가 없어 dsp 섹션의 Energy_Saving_Detail 각주에 병기했다.

**Power_Factor**: D25 소스(OM, SPS) 어디에도 D80의 "above 0.9" 같은 명시적 역률 수치가 없어 null 처리했다. PFC 기능 자체는 탑재되어 있음(PSU_Type 참조)이 확인되나 구체적 역률 수치는 미확인이다.

**[3] PSU_Mains_Rated_Power**: D25 OM/SPS의 첫 페이지 요약란에 "Peak output"이라는 항목명이 있으나(위 [2] 참조), PDF 레이아웃 추출 뒤섞임으로 인해 이 항목에 정확히 대응하는 수치를 자신 있게 분리해낼 수 없었다 — OM Chapter 3.1의 상세 전류표에도 "Peak output"이라는 명칭의 단일 대표값 행이 별도로 존재하지 않는다(Sine wave max 1s 조건의 채널별 값만 존재, 총 메인즈 입력 전력으로 환산되지 않음). 무결성 원칙(원문에 없는 값을 추정하여 채우지 않음)에 따라 null 처리 — D90의 "Peak output 3650W"처럼 이 필드에 대응하는 명확한 단일 수치를 D25 소스에서 재확인할 필요가 있다.

### 주석 및 출처 (Notes & Sources)

**출처**: OM p.9 "3 Technical specifications" (최우선) / SPS 2p

**측정 조건**: Frequency_Response는 -1 dB 허용오차, Linear mode 기준(OM/SPS 일치). THD_N/Crosstalk은 unweighted, 20-20 kHz 대역, "4 x 250 W into 8/4 Ω" 조건(long term 정격 기준, OM/SPS 일치).

**[1] Frequency_Response**: D90과 완전히 동일한 값(35-25000 Hz) — 같은 세대 플랫폼으로 판단.

**[2] THD_N 표기 방식 (D90과 동일한 dB/% 병기 방식)**: D25의 OM/SPS는 "THD+N/Crosstalk < -86 dB/0.005%"(8 Ω), "THD+N (unweighted, 20-20 kHz) < -83 dB/0.007%"(4 Ω)로 D90과 동일하게 dB를 주표기, %를 괄호 병기하는 방식을 사용한다. D90(< -80 dB/-74 dB)보다 D25가 더 낮은 THD 수치(더 우수한 성능)를 보이며, 이는 세대/모델 간 실제 사양 차이로 판단(충돌 아님).

**[3] Channel_Separation**: OM/SPS "Crosstalk (20-20 kHz) < -70 dBr"로 D80/D90과 완전히 동일한 수치("less than -70 dBr"). D80 각주에서 설명한 부호/기준 차이(L-Acoustics의 상대적 "greater than" 표기와 반대 방향)가 D25에도 동일하게 적용되며, 원문 그대로 보존했다.

**Output_Dynamic_Range, Noise_Level, Damping_Factor**: D25 소스에는 D80/D90과 마찬가지로 이 세 Key에 정확히 대응하는 스펙이 확인되지 않는다. SN_Ratio_Analog_Input/SN_Ratio_Digital_Input도 D90과 동일하게 D25 소스에 단순 dBr 상대값 형식의 S/N ratio 항목이 없어 null 유지 — D25의 실제 값은 D90과 동일한 절대 노이즈 전압(uV RMS) 기준 데이터 구조이며 아래 Output_Noise_Dynamic_Range_Detail Key에 기록했다.

**[4] Output_Noise_Dynamic_Range_Detail**: OM/SPS "Output noise (BW 20 kHz)/dynamic range (BW 20 kHz, reference 180 Vpk)" 조건, Analog/Analog A-weighted/Digital-Milan/Digital-Milan A-weighted 4개 조건 병기 — D90과 동일한 데이터 구조. **소스 자체의 수치 불일치 발견(v1.1 정정)**: 이 reference 전압(180 Vpk)은 D25 자신의 Max_Output_Voltage_Peak(120V, amplification 섹션)와 일치하지 않는다 — D90의 경우 reference 210 Vpk가 D90의 Max_Output_Voltage_Peak(210V)와 정확히 일치했던 것과 대조적이다. 180 Vpk는 오히려 D40의 Max_Output_Voltage_Peak(180V)와 일치하여, D25 OM/SPS의 이 항목이 D40 문서 템플릿에서 복사되며 갱신되지 않은 오기일 가능성이 있다(D40 파싱 중 D40 자신은 reference/Max가 정합적임을 확인하며 발견, D40_v1.0.md 각주 참조). 다만 이는 추정이며 원문 값(180 Vpk, 그리고 그에 연동된 절대 uV/dB 수치)을 임의로 변경하지 않고 그대로 채택했다 — 이 불일치 자체를 각주로 명시적으로 남긴다. v1.0에서는 이 각주가 "reference 전압이 Max_Output_Voltage_Peak와 일치한다"고 잘못 기술되어 있었으며, v1.1에서 정정했다(수치 자체는 변경 없음, 각주 서술만 수정).

## 버전 변경 이력

v1.0: Phase 3-1 동일 브랜드(d&b audiotechnik) 세 번째 제품 최초 파싱(D80, D90에 이어). OM(Reference Manual 1.2, 71p), SPS(Datasheet 1.1, 4p — 마지막 페이지에 Architectural specifications 절 포함) 2개 소스를 교차 검증하여 마스터 스키마 구축. D90_v2.0.md를 스켈레톤으로 복제(SKILL v1.11 "신규 제품 스켈레톤 재사용 규칙") — D25는 D90과 완전히 동일한 세대 플랫폼(Milan Endstation, 4.3" 터치스크린, LoadMatch/ArrayProcessing/System check/AmpPresets/EcoMode/Fallback/Override/AutoStandby/AutoWakeup)임이 확인되어 대부분의 Key/Unit/각주 구조가 그대로 재사용되었고, 실제 값(전력 정격, CMRR, 팬 개수, 무게 등)만 D25 고유 수치로 교체했다. 이미 구조 개편 v2.0(d80_specific/d90_specific 폐지, BTL/PBTL 10->4 통합, network 섹션 신설, D80_v2.0.md/D90_v2.0.md에서 확정)이 반영된 최신 스키마로 최초 작성되어 구식 섹션이 아예 존재하지 않는다. 신규 Key 1개(Max_Output_Current_Peak, amplification) 발견 — OM/SPS "Maximum output voltage/current 120 Vpeak/35 Apeak"에서 확인되었으나 기존 스키마에 대응 Key가 없었다. 기존 9개 파일(D80, D90, LA12X, LA7.16, LA1.16i, LA2Xi, LA4X, LA7.16i)의 amplification 섹션에 이 Key를 `null`로 소급 반영이 필요하다(세션 후속 조치 대상). PDF 레이아웃 추출 상 다단 표가 뒤섞이는 문제(전원 소비 요약란의 "Standby/Idling/Peak output" 수치)를 OM Chapter 3.1의 상세 상태별 전류표(전압×전류×역률 산술로 자체 검증 가능)로 교차검증하여 신뢰도 있는 값만 채택한 사례를 power_supply 섹션 각주에 상세 기록.

v1.0 to v1.1: (D40 파싱 중 발견된 각주 오류 정정) D40을 파싱하며 D40 자신의 Output_Noise_Dynamic_Range_Detail reference 전압(180 Vpk)이 D40의 Max_Output_Voltage_Peak(180V)와 정확히 일치함을 확인하는 과정에서, D25의 동일 항목 각주가 "D25의 reference 180 Vpk가 D25 자신의 Max_Output_Voltage_Peak(120V)와 일치한다"고 잘못 기술되어 있음을 발견했다(실제로는 불일치 — 180Vpk는 D25가 아닌 D40의 Max와 일치). audio_performance 섹션 각주 [4]를 정정하여 이 불일치를 명시적으로 기록하고, D25의 SPS/OM이 D40 문서 템플릿에서 복사되며 갱신되지 않았을 가능성을 각주에 남겼다. Value(180 Vpk 및 연동된 uV/dB 수치)는 원문 그대로 변경하지 않았다 — 각주 서술만 수정. **버저닝 절차 참고**: 이 수정은 최초에 이미 git 커밋된 D25_v1.0.md를 직접 Edit하는 방식(Surgical Versioning Protocol 위반)으로 시작되었으나, 커밋 이전에 발견하여 git checkout으로 되돌린 뒤 정식 절차(v1.0 복제 -> Archive 이관 -> v1.1에서만 수정)로 재작업했다.

v1.1 to v1.2: (5D 파싱 중 발견된 신규 스키마 확장의 양방향 동기화 반영 — null-only 추가, 실제 D25 데이터 변경 없음) 신규 제품 5D(d&b Dante 설치용 라인, 5D_v1.0.md) 파싱 과정에서 기존 스키마 어디에도 대응 개념이 없는 두 가지 확장이 발견되어, SKILL v1.11 "완벽한 양방향 스키마 동기화" 원칙에 따라 D25에도 소급 반영했다: (1) **input_dante 신규 섹션 추가**(input_avb 섹션 직후, network 섹션 직전 — 5D_v1.0.md와 동일한 섹션 순서) — Dante_RX_Channels, Dante_Sampling_Rates, Dante_SRC, Dante_Redundancy, Dante_Network_Flows, Dante_Latency, Dante_IP_Architecture 7개 Key를 전부 null로 추가(D25는 Milan Endstation을 사용하며 Dante를 사용하지 않으므로 비적용).
